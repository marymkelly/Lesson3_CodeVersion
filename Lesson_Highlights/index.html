Version Control and Git

Version Control

Version Control Systems allow us to track file changes. They are very useful for writers, designers, and developers. There are local, centralized, and distrubuted version control systems.

Git

Git is a distributed version control system. Created by the Linux community, the goals for the DVCS were as follows: speed, simple design, strong support for non-linear development, fully distributed, and able to handle large projects like the Linux kernel efficiently (speed and data size). It was completed in 2005.

Git Commands

Git has quite a few commands. For example "git init" initializes or creates a file; "git status" will show you what files have been modified and ready to be added; "git add" will allow you to add new content for committment; "git commit" will allow you to finalize putting content up; "git push" will allow you to push everything up to the remote repository.
